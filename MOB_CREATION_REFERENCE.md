# Референс: Правила Создания Мобов

## Цель
Этот документ фиксирует единый стандарт добавления новых мобов в проект.

## 1. Обязательные поля моба
При создании моба должны быть заполнены:
- `name` (уникальное имя в рамках `location_id`)
- `level`
- `health_points`, `max_health_points`
- `damage_min`, `damage_max`
- `armor_class`
- `experience_reward`, `gold_reward`
- `location_id`
- `mob_type` (например: `animal`, `humanoid`, `giant`)
- `aggression_type` (`passive` или `aggressive`)
- `respawn_time`
- `position_x`, `position_y`, `position_z` (если моб участвует в системе позиционирования)

## 2. Тиры и вариации
Для линии развития моба (пример: волки) использовать минимум 3-4 варианта:
- Молодой/Слабый
- Базовый
- Матерый/Ветеран
- Вожак (мини-босс)

Каждый следующий тир должен увеличивать:
- уровень
- запас здоровья
- урон
- награду (опыт/золото)

## 3. Привязка к зонам
Каждый моб должен быть связан с `mob_spawn_zones` через `mob_zone_spawns`.

Требования:
- мобы стартового контента: зоны `hunting`
- ресурсные зоны `resource` не использовать как боевые по умолчанию
- в `city` боевые мобы не спавнятся

## 4. Лут
У каждого боевого моба должна быть корректная `loot_table_id`.
Для новых линеек мобов:
1. создать/переиспользовать запись в `loot_tables`
2. добавить дроп в `loot_items`
3. проверить диапазоны `drop_chance`, `min_quantity`, `max_quantity`

## 5. Баланс стартовых мобов
Для стартовых зон ориентиры такие:
- T1: легкий фарм без обязательного кайта
- T2: умеренная сложность, требует базовой экипировки
- T3+: заметная угроза, желательно в группе или с прокачкой

## 6. Проверка после добавления
После добавления моба обязательно проверить:
- моб отображается в `world/zones/{location_id}`
- моб появляется в зоне через `mob_zone_spawns`
- корректно участвует в бою
- лут выпадает из ожидаемой таблицы
- уровень и награда соответствуют тиру

## 7. Именование
Рекомендуемый формат:
- `<Тир> <Базовый вид>` (например: `Молодой волк`, `Матерый волк`)
- уникальность и читабельность важнее декоративности
