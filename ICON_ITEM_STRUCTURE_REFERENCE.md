# Референс: Структура Иконок Предметов

## Базовый путь
Все иконки предметов хранятся в:
- `frontend/public/icons/items`

## Правило структуры
Иконки должны раскладываться по типам предметов в отдельные подпапки.
Пример уже принятой структуры:
- `frontend/public/icons/items/Weapon/One handed sword`

## Целевое расширение
Для новых категорий создавать отдельные директории, например:
- `frontend/public/icons/items/Armor/...`
- `frontend/public/icons/items/Consumables/...`
- `frontend/public/icons/items/Resources/...`

## Нейминг
- имя файла иконки должно быть стабильным и предсказуемым
- предпочтительно использовать единый стиль (`snake_case` или `kebab-case`) внутри одной категории

## Интеграция
Если иконка не находится автоматически, добавлять явный резолвер в фронтенд-маппинг (`iconResolvers.js`) до внедрения общего индексатора.

## Важно
Эта структура считается базовым стандартом проекта для оружия, брони, расходников, ресурсов и других типов предметов.
